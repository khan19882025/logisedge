Help & Tips
Rule Configuration Tips
  • Choose fields that are most likely to identify duplicates
  • Set appropriate similarity threshold (70-90% recommended)
  • Use priority to control rule execution order
  • Test rules on small datasets first
Important Notes
  • Rules are processed in priority order
  • Higher thresholds reduce false positives
  • Auto-merge should be used carefully
  • Test thoroughly before production use
Best Practices
  • Start with simple rules and refine
  • Monitor false positive rates
  • Use descriptive names and descriptions
  • Review merge results regularly
