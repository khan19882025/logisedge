Rule Types Guide
Validation Rules

Check data against defined criteria (format, range, business rules)

Transformation Rules

Convert data from one format to another (standardization, formatting)

Deduplication Rules

Identify and handle duplicate records based on specified criteria

Enrichment Rules

Add missing data or enhance existing records with additional information

Best Practices
  • Start with simple rules and gradually add complexity
  • Test rules on small datasets before applying to production
  • Use descriptive names and clear descriptions
  • Set appropriate timeouts for complex operations
  • Document any custom logic in the description
